# Epic 10: Traces, Evals, Hardening, Remote Readiness, and v1 Gate

**Objective:** Make the system durable, inspectable, and ready for serious continuation. After this epic, the system has evaluation infrastructure, comprehensive trace coverage, performance hardening, and is ready for a v1.0 release.

**Why it exists:** Feature completeness is not production readiness. Without evals, the system has no quality metrics. Without trace coverage, debugging is guesswork. Without hardening, edge cases cause crashes. Without a release gate, there's no confidence the system works end-to-end.

**What it unlocks:** v1.0 release and all future development

**Dependencies:** All prior epics (1-9)

**Phase:** 4

---

## Scope

### Included
- Eval framework with YAML spec format
- Compilation quality eval suite
- Retrieval and citation eval suite
- Trace coverage audit across all commands
- Error handling hardening
- Performance profiling against documented targets
- CLAUDE.md final update for v1 operations
- Status document update
- Test coverage gap closure to targets
- npm package preparation
- v1.0 release checklist and gate
- v1.0 release cut

### Excluded
- Remote/cloud features (Phase 5+)
- Multi-user collaboration (future)
- Plugin system (future)
- Vector search (deferred per blueprint)

## Beads

### E10-B01: Eval Framework and Spec Format
- **Depends on:** E4-B01, E6-B01
- **Produces:** `packages/kernel/src/evals/runner.ts`, `evals/` directory with sample eval specs. YAML spec format: eval name, type, input, expected outputs, scoring rubric. `ico eval run` discovers and executes specs.
- **Verification:** `ico eval run` finds and executes specs. Results show pass/fail with scores. Sample evals test compilation and retrieval.

### E10-B02: Compilation Quality Eval Suite
- **Depends on:** E10-B01, E6-B02 through E6-B07
- **Produces:** `evals/compilation-quality/` with specs: (1) Summarize eval (key claims captured?), (2) Extract eval (right concepts?), (3) Synthesize eval (covers all sources?). 5+ criteria scored 1-5.
- **Verification:** Evals execute with scores. Reproducible. At least 3 specs.

### E10-B03: Retrieval and Citation Eval Suite
- **Depends on:** E10-B01, E7-B01 through E7-B04
- **Produces:** `evals/retrieval-citation/` with specs: (1) Retrieval accuracy (recall@5, precision@5), (2) Citation fidelity, (3) Answer grounding. Fixture questions with known-good answers.
- **Verification:** Evals execute. Retrieval measures recall/precision. Citation fidelity catches hallucinated citations.

### E10-B04: Trace Coverage Audit
- **Depends on:** All prior epics
- **Produces:** Trace coverage report. Fixes for missing traces. Walks through every command verifying: trace emitted, schema conformance, correlation_id grouping. Also verifies audit trail integrity — walks the hash chain from E1-B03/E3-B06 and reports any breaks (audit H4).
- **Verification:** Every CLI command emits at least one trace. All traces pass schema validation. Correlation IDs group related events. Hash chain walk reports zero breaks on clean workspace.

### E10-B05: Error Handling Hardening
- **Depends on:** All prior epics
- **Produces:** Error handling improvements: user-friendly messages everywhere, no unhandled promise rejections, SQLite transactions for multi-writes, file system error handling, Claude API error specificity. SIGINT handler testing — send SIGINT to subprocess mid-operation and verify workspace is left in a consistent recoverable state (audit M15). Disk-full simulation testing via mocked fs to verify graceful degradation (audit M3).
- **Verification:** Trigger each error category → user-friendly message. No unhandled exceptions. Integration tests for error paths. SIGINT mid-operation → consistent state. Disk-full mock → graceful error, no corruption.

### E10-B06: Performance Profiling and Optimization
- **Depends on:** All prior epics
- **Produces:** Performance report against targets: ingest <2s, compile topic <30s, ask <10s, render report <5s, lint <30s. Bottleneck identification. Optimizations: FTS5 indexing, SQLite connection, file I/O, prompt size. Large corpus scenario: 500+ synthetic sources generated by a data generator script to stress-test at scale (audit M1). Performance benchmarks directory (`benchmarks/`) tracking results over time.
- **Verification:** Each operation meets target on moderate corpus (50 sources, 30 compiled pages). Large corpus (500+ sources) completes without failure or degradation beyond 3x baseline. Before/after measurements for optimizations. Benchmark results recorded.

### E10-B07: CLAUDE.md and README.md Final Update for v1 Operations
- **Depends on:** All prior epics
- **Produces:** Updated CLAUDE.md: remove "Pre-implementation" language, document all working CLI commands, update architecture for deviations, add "Session Operations" section, troubleshooting tips. Also update README.md with current feature list, installation instructions, and quick-start guide (audit finding — no bead previously covered README updates). Document how the operator uses traces for context refinement per the learning model (audit M8 from arch).
- **Verification:** CLAUDE.md accurately describes working system. All commands listed. No stale references. Session ops actionable. README.md current and accurate. Trace-based context refinement documented.

### E10-B08: Status Document Update (006-OD-STAT)
- **Depends on:** All prior epics
- **Produces:** Updated `000-docs/006-OD-STAT-status.md`: current phase, all features, real metrics (coverage, CI pass rate, eval scores), decision log, risk assessment.
- **Verification:** Status accurate. Real metrics. No outdated information.

### E10-B09: Test Coverage Gap Closure
- **Depends on:** All prior epics
- **Produces:** Additional tests closing gaps. Targets: kernel 90%, compiler 80%, cli 70%, types 100%. Focus on: state machine, promotion rules, provenance, search, frontmatter validation. Add Unicode fixtures across all packages to verify i18n handling. Add migration runner edge case tests (audit L5).
- **Verification:** All packages at or above target. No critical path untested. Unicode fixtures pass across all packages. Migration edge cases covered.

### E10-B10: npm Package Preparation
- **Depends on:** All prior epics
- **Produces:** Publication-ready package.json. `ico` binary works from global install. prepublishOnly runs build+test+lint. npm pack + install verified. Create CONTRIBUTING.md with development setup, coding conventions, PR process, and testing instructions (audit L12).
- **Verification:** `npm pack` produces tarball. Global install makes `ico` available. `ico --version`, `ico --help`, `ico init test` all work. CONTRIBUTING.md exists and covers dev setup, conventions, PR process, testing.

### E10-B11: v1.0 Release Checklist and Gate
- **Depends on:** E10-B01 through E10-B10
- **Produces:** Release checklist: (1) CI passes, (2) evals pass, (3) coverage targets met, (4) docs updated, (5) CHANGELOG complete, (6) no critical beads open, (7) user journey walkthrough succeeds, (8) performance targets met. Go/no-go decision.
- **Verification:** Every item passes. User journey documented. Release approved.

### E10-B12: v1.0 Release Cut
- **Depends on:** E10-B11
- **Produces:** Version bump to 1.0.0. CHANGELOG entry. Git tag. GitHub Release. npm publication.
- **Verification:** GitHub Release exists. npm package available. `npm install -g intentional-cognition-os` works. `ico --version` shows 1.0.0.

## Exit Criteria

1. Eval framework works with compilation and retrieval/citation suites
2. Every system operation produces a schema-conformant trace
3. All error paths produce user-friendly messages
4. Performance meets documented targets
5. Test coverage meets targets (kernel 90%, compiler 80%, cli 70%)
6. CLAUDE.md and status doc reflect complete v1 system
7. npm package installs and runs correctly
8. v1.0 release cut with complete release notes
9. All prior integration tests remain green
10. README.md updated with current features and quick-start guide
11. CONTRIBUTING.md exists with development setup and contribution guidelines

## Risks / Watch Items

- **Coverage gap closure can produce low-value tests.** Mitigation: focus on critical paths, not line-count gaming.
- **Performance optimization rabbit hole.** Mitigation: measure first, only optimize misses.
- **npm package name availability.** Mitigation: verify early.
- **User journey walkthrough is the real test** — if any step fails, real work remains. Mitigation: don't rush the gate.
